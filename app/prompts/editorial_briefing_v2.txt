==============================================================
連續性處理（重要 — 由 thread / phase 結構驅動）
==============================================================

下方 `thread_groups` 已經把訊號**按故事線分組**，每組帶 thread context（known_background、do_not_repeat_points、continuation_prompt_hint）以及 phase tree（已開的敘事軸與狀態）。

判斷規則（必須照做）：

1. **`thread.do_not_repeat_points` 列出的內容禁止再寫**——昨日 briefing 已詳述。如果今日只有重複的訊號，不要為這條 thread 開 section；若有新進展，只寫 delta。

2. **`signal.is_background_repeat == true`**：W7 已標記這是重複舊內容、無新發展。**不要把這類訊號當成新 section 的主訊號**；最多在既有 section 用一句話帶過「同主題後續報導」。

3. **`thread.status` 與 phase status**：
   - phases 中有 `status: emerging` → 這是**新軸**（剛分出來），是今日真新進展，優先寫
   - phases 全是 `dormant` → 這條 thread 今天沒新東西，跳過
   - thread 有 `today_delta` 文字 → 那段就是 delta 摘要，sections 圍繞它寫

4. **`continuation_prompt_hint`**：是 W7 給的「該怎麼帶過背景」一句話，section 開頭可直接借用語感。

5. **延續主題的 section 必須**：
   - `is_continuation: true`
   - `continuation_note`：1-2 句，從 `known_background` / `do_not_repeat_points` 反向取「最少必要背景」（讀者沒聽昨日也聽得懂），但**不要重講昨日已詳述的細節**
   - `summary` 主力寫今日 delta（用 thread 的 today_delta + 各 signal 的 today_delta / what_happened）

6. **新主題（thread 第一次出現，或無 thread_id 的 ungrouped signal）**：
   - `is_continuation: false`
   - `continuation_note: ""`
   - 完整鋪陳

7. **`thread_mismatch_suspected == true`**（W7 phase 判斷此 signal 可能掛錯 thread）：**寫的時候要謹慎**，可寫但不要用它的 thread context 去推論——這條 signal 的 thread 分組不可靠。

8. **`duplicate_suspected == true`**（W7 phase 疑似 W4 漏抓的重複）：**不要為它單獨開 section**，當成既有 section 的補充來源即可。`adjudication_rationale` 會說明可能的原始 signal id。

> 註：`adjudication_decision` 是 W4 的 item↔signal 判斷（`same_event` / `same_thread` / `different_event`），與上述 W7 phase flags 不同；多數情況不需要直接根據 `adjudication_decision` 改變寫法，但若帶 `same_event` / `same_thread` 表示 W4 已確認延續關係，可加強 `is_continuation` 判斷。
